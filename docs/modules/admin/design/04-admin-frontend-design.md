## 0. 模块定位

`admin` 是一个独立的 Web 后台模块，集成到 `pipecat-server` FastAPI 服务中，负责：
- 提供 `/admin/*` HTML 页面路由（Jinja2 渲染）
- 托管 Jinja2 模板和静态资源
- **不包含任何业务数据逻辑**；数据通过调用 `pipecat-pipeline` 模块提供的 `/api/admin/*` 接口获取

**文件结构**：

```
pipecat-server/
├── admin_web.py          # 新增：FastAPI Router，/admin/* HTML 路由
└── templates/
    ├── admin/
    │   ├── base.html               # 公共布局（顶栏 + 侧边栏）
    │   ├── overview.html           # 概览页
    │   ├── conversations.html      # 对话列表页
    │   └── conversation_detail.html # 对话详情页
```

`main.py` 中注册 `admin_web` router（`include_router`），并挂载 `templates/` 目录（Jinja2）。

## 1. 设计概述

### 1.1 目标

为 pipecat-server 集成一个经典管理后台 UI，用于查看对话历史和性能指标，并为后续管理功能预留扩展位。

### 1.2 技术栈

| 组件 | 选型 | 说明 |
|------|------|------|
| 渲染方式 | Jinja2 服务端渲染 + fetch JSON | 首屏 HTML 由服务端生成，动态数据通过 fetch 调用 `/api/admin/*` 更新 |
| 样式 | Tailwind CSS（CDN） | 无构建步骤，适合本地工具 |
| 图表 | 纯 CSS 进度条 | 耗时时间轴用 CSS flexbox 实现，无需引入 JS 图表库 |
| 字体/图标 | Heroicons（CDN SVG inline） | 轻量图标 |

### 1.3 设计约束

- 无响应式要求（桌面浏览器即可）
- 无无障碍强制要求（内部开发工具）
- 无登录/鉴权

## 2. 整体布局

所有页面继承 `base.html`，采用经典三段式管理后台布局：

```
┌─────────────────────────────────────────────────────────────┐
│  🎙 VoiceMask Admin                          [顶部导航栏]    │
├──────────────┬──────────────────────────────────────────────┤
│              │                                              │
│  [侧边栏]    │  [主内容区 - block content]                  │
│  ─────────   │                                              │
│  📊 概览     │                                              │
│  💬 对话记录 │                                              │
│              │                                              │
│  ─────────   │                                              │
│  （预留）    │                                              │
│              │                                              │
└──────────────┴──────────────────────────────────────────────┘

**顶部导航栏**：品牌名 + 当前服务器地址/状态（静态展示）

- 预留分隔线，后续功能模块追加到分隔线下方

│                                                             │
│  耗时分解                                                    │
│  ┌──────────────────────────────────────────────────────┐   │
│  │  环节       首包时间    总计时间                       │   │
│  │  ASR        340 ms     340 ms                        │   │
│  │  LLM        490 ms    1200 ms                        │   │
│  │  TTS        220 ms     890 ms                        │   │
│  │  ─────────────────────────────                       │   │

│                                                             │
│  时间轴（首包时间占比）                                       │
│  ┌──────────────────────────────────────────────────────┐   │
│  │  ASR  [████████████████░░░░░░░░░░░░░░░░░]  340ms    │   │
│  │  LLM  [░░░░░░░░░░░░░░░░████████████████░░]  490ms   │   │

│  └──────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
```

**时间轴说明**：
- 横轴总宽度 = 整体首包时间（e2e_ttfa_ms）
- 每个环节的色块宽度 = 该环节首包时间 / e2e_ttfa_ms × 100%
- 色块起始位置 = 上一环节结束位置（首尾衔接）
- 纯 CSS 实现（`width: calc(...)` + inline style）

## 6. 路由与页面交互

```
/admin
  │ 页面加载后 fetch /api/admin/stats
  │ 渲染统计卡片 + 最近对话
  │ 点击任意对话行
  ▼
/admin/conversations/{id}

/admin/conversations
  │ 页面加载后 fetch /api/admin/conversations?page=1
  │ 渲染表格 + 分页
  │ 点击上一页/下一页 → 更新 URL query param，重新 fetch
  │ 点击任意行
  ▼
/admin/conversations/{id}
  │ 服务端直接渲染完整数据（无需 fetch）
  │ 点击"返回列表"
  ▼
/admin/conversations
```

## 7. 状态管理

本项目为服务端渲染 + 简单 fetch，无需客户端状态管理框架。

| 状态 | 管理方式 |
|------|---------|
| 当前页码 | URL query param `?page=N` |
| 统计数据 | fetch 后写入 DOM（innerHTML） |
| 侧边栏高亮 | Jinja2 模板根据当前路由条件渲染 `active` class |

## 8. 数据获取策略

| 页面 | 方式 | 加载态 | 空态 | 错误态 |
|------|------|--------|------|--------|
| 概览 | 页面加载后 fetch `/api/admin/stats` | 卡片显示 `--` | 卡片显示 `0` / `N/A` | 显示 `获取失败，请刷新` |
| 对话列表 | 页面加载后 fetch `/api/admin/conversations` | 表格显示 skeleton 行 | 显示「暂无对话记录」 | 显示错误提示 |
| 对话详情 | 服务端 Jinja2 渲染 | — | — | 404 页面 |

## 9. 影响评估

| 影响项 | 评估 |
|--------|------|
| 后端 API 依赖 | 依赖 `pipecat-pipeline` 模块提供的 `/api/admin/*` 全部端点（见后端设计文档） |
| 现有 `/ws` 路由 | 无影响 |
| 模块独立性 | `admin_web.py` 不引用 pipecat-pipeline 内部代码，仅通过 HTTP 接口解耦 |
| 包体积 | Tailwind CDN 约 30KB gzip，无本地 bundle |
| 首屏加载 | 服务端渲染，首屏无空白等待 |
