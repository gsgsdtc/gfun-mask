# Feat-02: ESP32 音频采集与 Opus 编码传输

> 更新日期：2026-03-05
> 来源：Epic-01 Phase 2 — 音频管道建立
> 优先级：P0（核心功能）
> 前置依赖：feat-01（BLE L2CAP 通道验证）

## 1. 功能概述

### 1.1 背景

feat-01 已验证 ESP32 与 iOS 之间的 BLE L2CAP 通道可正常传输数据。下一步需要建立完整的音频管道：ESP32 通过麦克风采集音频、编码压缩后传输到 iOS，iOS 端接收并保存为可播放的音频文件。

**硬件阶段说明**：
- **当前阶段**：使用 ESP32-S3-BOX-Lite 官方开发板进行功能验证（该开发板已集成麦克风）
- **后续阶段**：自研硬件（硬件规格可能与开发板不同，需要考虑软件层面的兼容性设计）

### 1.2 目标

实现 ESP32 端音频采集、Opus 编码、BLE L2CAP 传输，以及 iOS 端音频接收、文件保存和播放功能。用户可以在 iOS App 中查看历史录音、播放音频内容，验证音频管道的正确性。

### 1.3 非目标

- 不涉及 VAD（语音活动检测）
- 不涉及 Pipecat 后端集成
- 不涉及实时语音对话
- 不涉及后台常驻与进程保活
- 不涉及动态音频码率调整

## 2. 用户场景

### 场景 1: 开始录音

- **角色**：用户
- **触发**：用户在 iOS App 点击"开始录音"按钮
- **流程**：
  1. iOS 通过 BLE 向 ESP32 发送"开始录音"指令
  2. ESP32 启动麦克风采集音频
  3. ESP32 实时将音频数据编码为 Opus 格式
  4. ESP32 通过 BLE L2CAP 信道持续发送音频帧
  5. iOS 接收音频帧并缓存
- **结果**：App 显示"录音中"状态，实时显示已录制时长

### 场景 2: 停止录音并保存

- **角色**：用户
- **触发**：用户在 iOS App 点击"停止录音"按钮
- **流程**：
  1. iOS 通过 BLE 向 ESP32 发送"停止录音"指令
  2. ESP32 停止麦克风采集，发送"录音结束"标记
  3. iOS 将缓存的 Opus 数据封装为完整音频文件
  4. iOS 将音频文件保存到本地
- **结果**：App 显示"已保存"提示，录音列表中出现新记录

### 场景 3: 播放历史录音

- **角色**：用户
- **触发**：用户在录音列表中点击某条录音记录
- **流程**：
  1. App 进入播放界面，显示录音信息（时长、创建时间）
  2. 用户点击播放按钮
  3. App 解码 Opus 音频并播放
- **结果**：用户听到录制的音频内容

### 场景 4: 查看录音详情

- **角色**：用户
- **触发**：用户在录音列表中查看某条录音的详情
- **流程**：
  1. App 显示录音元数据：文件名、时长、文件大小、创建时间、编码格式（Opus）
  2. App 提供删除功能
- **结果**：用户了解录音信息，可选择删除

## 3. 验收标准

### 功能验收

- [ ] ESP32 能通过麦克风采集音频（采样率 16kHz，单声道）
- [ ] ESP32 能将采集的音频实时编码为 Opus 格式
- [ ] ESP32 能通过 BLE L2CAP 持续传输 Opus 音频帧
- [ ] iOS 能接收完整的音频帧序列，无丢帧或乱序
- [ ] iOS 能将接收的 Opus 数据保存为可播放的音频文件
- [ ] iOS 能播放保存的录音，音质清晰无明显杂音或断续
- [ ] iOS 录音列表能显示所有历史录音
- [ ] iOS 能显示录音的时长、大小、创建时间等信息
- [ ] iOS 能删除指定录音

### 非功能验收

- [ ] **音频质量**：Opus 编码码率 ≥ 16kbps，能清晰识别语音内容
- [ ] **传输稳定性**：连续录音 60 秒以上无中断、无丢帧
- [ ] **存储效率**：1 分钟录音文件大小 ≤ 1MB（目标约 600KB @ 16kbps）
- [ ] **响应速度**：从点击"开始录音"到实际开始采集 ≤ 500ms
- [ ] **兼容性**：iOS 版本 ≥ 16.0；ESP-IDF 版本 ≥ 5.x
- [ ] **硬件可移植性**：麦克风驱动层抽象，支持后续适配自研硬件（不与 ESP32-S3-BOX-Lite 特定 GPIO 硬编码耦合）

## 4. 约束条件

### 4.1 音频约束

- **音频格式**：必须使用 Opus 编码（低延迟、高压缩比，适合实时传输）
- **采样率**：16kHz（语音场景的标准采样率）
- **声道**：单声道（语音场景无需立体声）
- **传输通道**：复用 feat-01 验证的 BLE L2CAP CoC 信道

### 4.2 硬件约束

- **当前开发硬件**：ESP32-S3-BOX-Lite（官方开发板，已集成 ES7210 麦克风）
- **后续自研硬件**：硬件规格可能与开发板不同，软件设计需考虑：
  - 麦克风驱动层的抽象，便于后续适配不同型号
  - I2S 配置参数可配置化（采样率、增益等）
  - 避免与开发板特定 GPIO 硬编码耦合

### 4.3 平台约束

- **iOS 权限**：App 需声明蓝牙使用权限（音频文件保存在本地，无需麦克风权限）
- **文件格式**：iOS 保存的音频文件格式需支持 Opus（如 .opus 或 .ogg）
